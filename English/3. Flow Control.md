# Flow Control

This guide presents the mechanisms used to direct the logical flow of your scripts. Mastering conditions and loops is essential for creating reactive, intelligent, and performance-optimized FiveM scripts.

## Understanding Conditional Logic (if, elseif, else)

Conditional structures allow your script to make decisions based on context (e.g., "If the player is in a vehicle, then...").

### Why the order of conditions matters
Lua checks conditions sequentially. As soon as a condition evaluates to `true`, it executes the associated code and **ignores** the rest of the `if` block.

```lua
local speed = GetEntitySpeed(GetVehiclePedIsIn(PlayerPedId()))

if speed > 50.0 then
    -- If speed is 60, only this block executes
    print("Critical Speed")
elseif speed > 20.0 then
    -- This block will NEVER be reach if speed is greater than 50
    print("Moderate Speed")
else
    print("Low or zero speed")
end
```

### Table of Logical Operators

| Operator | Pedagogical Meaning |
| :--- | :--- |
| `==` | Checks for strict equality. |
| `~=` | Checks if two values are different. |
| `and` | All conditions must be true. |
| `or` | At least one condition must be true. |
| `not` | Inverts the result (true becomes false). |

## Loops: Working at Scale

Loops are used to process lists of data or to maintain repetitive actions (like a self-updating HUD).

### Iterating Tables: pairs vs ipairs
This is a crucial distinction in Lua:

1.  **ipairs**: Used for ordered numerical arrays {1, 2, 3}. More performant but stops at the first missing index.
2.  **pairs**: Used for dictionaries (named keys). Allows traversing the entire table, regardless of order.

```lua
local inventory = {"Bread", "Water", "Medkit"}

for index, item in ipairs(inventory) do
    print("Slot " .. index .. ": " .. item)
end
```

### The While Loop: Maximum Vigilance
It runs as long as a condition is true. In FiveM, it is often used in threads to monitor a player's state.

```lua
local isAlive = true
while isAlive do
    -- IMPORTANT: Without Wait(), this loop will freeze your Windows/FiveM
    Wait(1000) 
    -- State check here
end
```

## Framework Integration

### ESX Example: Filtering Inventory
We use `pairs` because the ESX inventory is often a complex table of objects.

```lua
local inventory = xPlayer.getInventory()

for _, item in pairs(inventory) do
    -- We check if the count is greater than 0
    if item.count > 0 then
        print("Player has " .. item.count .. "x " .. item.label)
    end
end
```

### QB-Core Example: Multi-Criteria Check
Using `and` and `or` operators for complex permissions.

```lua
local PlayerData = QBCore.Functions.GetPlayerData()

-- "If the player is a cop OR an EMS AND is on duty"
if (PlayerData.job.name == 'police' or PlayerData.job.name == 'ambulance') and PlayerData.job.onduty then
    -- Access allowed to the shared locker
end
```

> [!CAUTION]
> **The Infinite Loop Trap**: If you use `while true do` without a `Wait()` inside, your script will demand 100% of your CPU power instantly, causing a total game freeze.
