# Tables and Metatables

Tables are the only complex data structure in Lua. By mastering tables, you master 80% of the language. They serve every purpose: from a simple shopping list to the complex player object of a framework.

## The Two Faces of the Table

A table can be used in two primary ways, often mixed together in FiveM.

### 1. The Array
An ordered list of elements identified by a number (index).
*   **Vital Reminder**: Indexing starts at **1**.
*   **Usage**: Player lists, item inventories, coordinate sets.

```lua
local vehicles = {"T20", "Zentorno", "Adder"}

-- Pedagogical Manipulation
table.insert(vehicles, "Panto") -- Adds to the end of the list
table.remove(vehicles, 1)        -- Removes "T20" (others shift down)
print(#vehicles)                 -- The '#' operator gives the length (now 3)
```

### 2. The Dictionary (Key-Value)
A collection of elements identified by a name (key).
*   **Usage**: Player data, script configuration, vehicle properties.

```lua
local playerAccount = {
    name = "John",
    money = 5000,
    job = "Unemployed"
}

print(playerAccount.money) -- Direct access via the dot notation
```

## Metatables: Modifying Behavior

A metatable is a "rule table" that you attach to a normal table to change how it reacts to certain operations (addition, missing indexing, etc.).

### Missing Indexing (__index)
This is the most used metamethod. It allows you to define a "backup table" if you look for a key that doesn't exist in the main table.

```lua
local standardVehicle = { engine = "V6", color = "Black" }
local myCustomCar = { color = "Red" }

-- If a value isn't found in 'myCustomCar', look in 'standardVehicle'
setmetatable(myCustomCar, { __index = standardVehicle })

print(myCustomCar.color)  -- Output: "Red" (found locally)
print(myCustomCar.engine) -- Output: "V6" (inherited via the metatable)
```

## Framework Integration

### ESX Example: The Inventory
The ESX inventory is a table of dictionaries. Learning to traverse it is essential.

```lua
local inventory = xPlayer.getInventory()

-- 'item' is a small table containing {name, label, count, etc.}
for _, item in pairs(inventory) do
    if item.count > 0 then
        -- We use the keys of the sub-table
        print("Player has " .. item.count .. " " .. item.label)
    end
end
```

### QB-Core Example: Complex Data
QB-Core uses nested tables for character metadata.

```lua
local PlayerData = QBCore.Functions.GetPlayerData()

-- Accessing a value in a table stored inside another table
local hungerLevel = PlayerData.metadata["hunger"]

if hungerLevel < 10 then
    print("Player is hungry!")
end
```

## Advanced Table API Functions

| Function | Pedagogical Usage |
| :--- | :--- |
| `table.concat(t, sep)` | Merges a list of text into a single string (useful for logs). |
| `table.unpack(t)` | Transforms a list of values into arguments for a function. |
| `table.sort(t)` | Sorts elements (alphabetically or numerically). |

```lua
local coords = {100.5, 20.0, 30.0}
-- unpack(coords) is equivalent to writing 100.5, 20.0, 30.0
SetEntityCoords(ped, table.unpack(coords))
```

> [!CAUTION]
> **The Nil Trap**: If you insert `nil` into the middle of an indexed array, the `#` operator (length) will stop at that point and ignore the rest of the array. For lists with gaps, use a `pairs` loop to count elements manually.
