# FiveM Natives and API Functions

This guide focuses on the core functions (Natives) provided by FiveM to interact with the GTA V engine. Understanding these functions is essential for manipulating the world, players, and user interfaces.

## What is a "Native"?

Natives are low-level functions coded by Rockstar Games or the FiveM team. They are the building blocks of every script.

*   **Standalone Call**: You call the native directly (e.g., `GetEntityCoords`).
*   **Framework Call**: You use a "wrapper" function created by ESX or QB-Core, which internally calls several natives to simplify your task.

### The Essential Reference
All documented natives are listed here: [docs.fivem.net/natives/](https://docs.fivem.net/natives/). This is every developer's dictionary.

## Command Management (RegisterCommand)

Use `RegisterCommand` to create interactions via the chat or the F8 console.

### Why the 'source' parameter?
On the server side, `source` is a unique identifier for the player who triggered the command. **Always store it locally** at the very beginning of your function.

```lua
RegisterCommand("pos", function(source, args, rawCommand)
    -- 'source' is the server ID of the player
    local ped = PlayerPedId()
    local coords = GetEntityCoords(ped)
    print("Coordinates sent to F8 console")
end, false)
```

## Events: The Global Network

Events allow information to travel between the **Server** (the central brain) and the **Client** (each individual player).

### TriggerServerEvent vs TriggerClientEvent
*   **TriggerServerEvent**: The client asks the server for something (e.g., "I want to buy this").
*   **TriggerClientEvent**: The server instructs one (or all) client(s) to do something (e.g., "Display this message on screen").

```lua
-- Client Side
TriggerServerEvent("resourceName:checkLevel")

-- Server Side
RegisterNetEvent("resourceName:checkLevel", function()
    local _source = source -- Secure the source
    print("Player " .. _source .. " is checking their level")
end)
```

## Optimized Math: Vectors

A `vector3` is not just a list of three numbers (x, y, z); it is a **mathematical object**.

```lua
local ped = PlayerPedId()
local coords = GetEntityCoords(ped)

-- SIMPLIFIED distance calculation
local destination = vector3(100.0, 200.0, 30.0)
-- Using the '#' operator on subtracted vectors yields the distance in meters.
local distance = #(coords - destination)

if distance < 5.0 then
    -- Player is very close
end
```

## Framework Integration: Notifications

Rather than using complex GTA V graphical natives, use the pre-built functions provided by your framework.

### Comparison by Framework
```lua
-- ESX: Displays a standard notification in the bottom left
ESX.ShowNotification("Your vehicle has been impounded")

-- QB-Core: Supports types (success, error, primary)
QBCore.Functions.Notify("Transaction successful", "success", 5000)

-- Standalone (Original GTA V)
BeginTextCommandThefeedPost("STRING")
AddTextComponentSubstringPlayerName("Simple message")
EndTextCommandThefeedPostTicker(false, true)
```

> [!IMPORTANT]
> **Client/Server Desync**: A native called on the client side might not be synchronized on the server. For permanent actions (e.g., giving an item), always route through a server event.
