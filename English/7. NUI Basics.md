# NUI Fundamentals (HUD & UI)

NUI (New User Interface) is an integrated web browser (Chromium) within FiveM. While Lua handles game logic, NUI manages all complex visual aspects: menus, HUDs, and graphical inventories.

## Why use a browser inside a game?

Lua is not designed for creating modern graphical interfaces (with rounded corners, animations, shadows). By using HTML, CSS, and JavaScript, you have access to the same tools as professional web developers to create stunning interfaces.

## The Bridge: Lua <-> JS Communication

NUI and Lua live in two separate worlds. To allow them to speak to each other, FiveM created a communication "bridge."

### 1. Sending Data (Lua to JS)
We use `SendNUIMessage`. Think of this as sending a letter to a recipient (the browser).

**Client (Lua)**:
```lua
-- We send a Lua table which is automatically converted to JSON
SendNUIMessage({
    type = "OPEN_MENU",
    player = "John Doe",
    money = 1500
})
```

**Interface (JavaScript)**:
JS "listens" to everything arriving via the `message` event.
```javascript
window.addEventListener('message', (event) => {
    // Data sent by Lua is found in event.data
    if (event.data.type === "OPEN_MENU") {
        document.getElementById("name").innerText = event.data.player;
        document.getElementById("my-ui").style.display = "block";
    }
});
```

### 2. Returning Data (JS to Lua)
The browser cannot "call" a Lua function directly. It must make a `POST` request to your resource name.

**Interface (JavaScript)**:
```javascript
fetch(`https://${GetParentResourceName()}/close_ui`, {
    method: 'POST',
    body: JSON.stringify({ reason: "quit_button" })
});
```

**Client (Lua)**:
We register a "callback" (a function waiting for the call).
```lua
RegisterNUICallback("close_ui", function(data, cb)
    print("UI closed because: " .. data.reason)
    SetNuiFocus(false, false) -- Return keyboard control to the game
    cb('ok') -- Always send a response, even if empty
end)
```

## Focus: Mouse and Keyboard

By default, when a UI is open, your keys still control your character (e.g., 'W' makes the player move forward instead of typing in a text field).

*   `SetNuiFocus(true, true)`: Enables the mouse cursor AND blocks game keys for NUI.
*   `SetNuiFocus(false, false)`: Returns all control to the game.

## Optimization Tips (FPS)

1.  **Visibility**: An invisible interface (`opacity: 0`) is still calculated by the CPU. Use `display: none` to completely unload the rendering when the UI is closed.
2.  **Frequency**: Don't update your HUD (hunger, thirst) every frame (`Wait(0)`). Updating once per second is more than enough and preserves performance.
3.  **Asset Size**: Your NUI images must be lightweight. A 4MB image will slow down the inventory opening for all players.

> [!IMPORTANT]
> **GetParentResourceName()**: Always use this function in JavaScript rather than hardcoding your folder name. This allows your script to work even if the resource folder is renamed.
