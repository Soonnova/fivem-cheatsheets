# Lua Syntax and Data Types

This guide details the core foundations of the Lua language within the specific context of FiveM development. Although lightweight, Lua is powerful and serves as the backbone for virtually all GTA V scripts. Understanding its nuances is the first step toward writing efficient code.

## Structure and Execution

A Lua script in FiveM is a plain text file with the `.lua` extension. Execution is **sequential**: the engine reads the file from the first line to the last. If a syntax error occurs at the beginning of the file, the entire script may fail to load.

### The "local" Keyword: Why is it vital?

This is one of the most critical concepts for a FiveM developer. In Lua, by default, every variable is **global** unless explicitly declared as **local**.

1.  **Performance**: Accessing local variables is faster for the computer because they are stored in temporary "registers" rather than in a global environment table.
2.  **Safety**: It prevents "Script A" from accidentally modifying a variable in "Script B" if they both happen to use the same name.

```lua
-- RECOMMENDED: Local scope
-- This variable only exists within this file or code block.
local welcomeMessage = "Welcome to the server"

-- AVOID: Global scope
-- This variable is accessible by ALL other scripts, leading to unpredictable bugs.
globalMessage = "This can cause conflicts"
```

### Comments: Documenting the "Why"

Comments are ignored by the game engine. They serve to explain your logic to yourself (six months from now) or to other developers.

```lua
-- Simple comment: useful for brief explanations.

--[[
    Multi-line comment:
    Ideal for describing the overall purpose of a script
    or temporarily disabling a large block of code.
--]]
```

## Fundamental Data Types

Lua automatically identifies the type of data you provide (dynamic typing). Here are the ones you will use daily:

| Type | Pedagogical Description | Practical Example |
| :--- | :--- | :--- |
| **Nil** | Represents the absence of a value. If you look for a variable that doesn't exist, Lua returns `nil`. | `local x = nil` |
| **Number** | Includes both integers (1, 2) and decimals (1.5). FiveM uses these for IDs, prices, and coordinates. | `local price = 500` |
| **String** | A sequence of characters (text). Always enclosed in quotes `" "` or `' '`. | `local job = "Police"` |
| **Boolean** | A binary value: either `true` or `false`. Essential for conditions. | `local isDuty = true` |
| **Table** | The universal container in Lua. It can act as a simple list or a complex object. | `local data = {}` |

### String Manipulation

To merge (concatenate) two pieces of text, Lua uses the `..` operator. This is a common operation for displaying personalized messages.

```lua
local firstName = "John"
local lastName = "Doe"
-- We manually add a space between the two variables
local fullName = firstName .. " " .. lastName 
print(fullName) -- Will output "John Doe" in the console
```

## Tables: The Heart of Lua

Tables are used everywhere in FiveM (player objects, vehicle configurations, item lists).

### The Array
An ordered list of elements identified by a number (index). **Crucial Note:** In Lua, we start counting at **1**, not 0 as in C# or Python.

```lua
local jobs = {"Police", "EMS", "Mechanic"}
-- Accessing the first element
print(jobs[1]) -- Will output "Police"
```

### The Dictionary (Object)
A collection where each value is associated with a "key" (a name).

```lua
local vehicle = {
    model = "t20",
    plate = "ABC 123",
    engineHealth = 1000.0
}
-- Data is accessed via the key name (using the "dot" notation)
print(vehicle.model) -- "t20"
```

> [!IMPORTANT]
> Mastering tables is the most rewarding step in your learning journey. Almost all complex data returned by FiveM or your frameworks (ESX/QB) are tables.
