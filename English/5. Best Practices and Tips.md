# Best Practices and Performance Optimization

Developing for FiveM isn't just about making a script work; it's about making it work **without affecting others**. A server might run 100 scripts; if each one consumes slightly too much, the game becomes unplayable (FPS drops).

## Readability: Writing for Humans

Code is read far more often than it is written. Clean code makes debugging and collaboration much easier.

### Naming Conventions
Choose a convention and stick to it. Clarity is more important than brevity.

| Element | Recommendation | Why? |
| :--- | :--- | :--- |
| **Variables** | `playerMoney` or `accountBalance` | Avoid `pm` or `b`. You should know what's inside without re-reading the whole script. |
| **Functions** | `validateLicense()` | Use action verbs. |
| **Events** | `script:client:openMenu` | Use prefixes to identify which script triggered the event. |

## Modularity: Don't Repeat Yourself (DRY)

The **DRY** principle consists of avoiding copy-pasting the same code in multiple places. Use `exports` to share logic between scripts.

```lua
-- In the 'utils' script (fxmanifest: exports { 'capitalize' })
function capitalize(text)
    return (text:gsub("^%l", string.upper))
end

-- In another script
local formattedName = exports['utils']:capitalize("john")
```

## Technical Optimization (The Resmon)

The `resmon` (Resource Monitor) measures the CPU time used by your script in milliseconds (ms).

### Localizing Natives
Accessing a global FiveM function (Native) is slower than accessing a local function. In intensive loops, **localize your natives**.

```lua
-- Top of the file
local GetEntityCoords = GetEntityCoords
local PlayerPedId = PlayerPedId

-- The loop will save precious microseconds in every iteration
while true do
    local coords = GetEntityCoords(PlayerPedId())
    Wait(0)
end
```

## Security: The Client is a Liar

This is the golden rule for any multiplayer developer. **Anything coming from the client can be faked by cheating software.**

### The Right Instinct
The server must always be the sole judge for critical actions (money, inventory, permissions).

```lua
-- Server Side (SECURE)
RegisterNetEvent("shop:buyItem", function(itemName)
    local _source = source
    local xPlayer = ESX.GetPlayerFromId(_source)
    local price = 500 -- The price is defined on the SERVER, not sent by the client
    
    if xPlayer.getMoney() >= price then
        xPlayer.removeMoney(price)
        xPlayer.addInventoryItem(itemName, 1)
    end
end)
```

## Error Handling: pcall

If part of your code might "crash" (e.g., reading a table that might be empty), use `pcall` (protected call).

```lua
local success, result = pcall(function()
    -- Risky code here
    return targetTable[1].name
end)

if not success then
    print("Whops, an error was averted: " .. result)
end
```

> [!TIP]
> **The Secret of 0.00ms**: A perfect script is one that "sleeps" (high Wait value) when the player isn't specifically interacting with it. Optimization is the art of doing nothing by default.
