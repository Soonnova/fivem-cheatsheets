# Tables et Metatables

Les tables sont l'unique structure de données complexe en Lua. En maîtrisant les tables, vous maîtrisez 80% du langage. Elles servent à tout : de la simple liste de courses à l'objet joueur complexe d'un framework.

## Les deux visages de la Table

Une table peut être utilisée de deux manières principales, souvent mélangées dans FiveM.

### 1. Le Tableau (Array)
Une liste ordonnée d'éléments identifiés par un numéro (index).
*   **Rappel Vital** : L'indexation commence à **1**.
*   **Usage** : Liste de joueurs, inventaire d'objets, liste de coordonnées.

```lua
local voitures = {"T20", "Zentorno", "Adder"}

-- Manipulation pédagogique
table.insert(voitures, "Panto") -- Ajoute à la fin de la liste
table.remove(voitures, 1)        -- Supprime "T20" (les autres se décalent)
print(#voitures)                 -- L'opérateur '#' donne la longueur (ici 3)
```

### 2. Le Dictionnaire (Key-Value)
Une collection d'éléments identifiés par un nom (clé).
*   **Usage** : Données d'un joueur, configuration d'un script, propriétés d'un véhicule.

```lua
local compteJoueur = {
    nom = "John",
    argent = 5000,
    job = "Chômeur"
}

print(compteJoueur.argent) -- Accès direct via le point
```

## Les Metatables : Modifier le comportement

Une metatable est une "table de règles" que vous attachez à une table normale pour changer sa façon de réagir à certaines opérations (addition, indexation manquante, etc.).

### L'indexation manquante (__index)
C'est la métaméthode la plus utilisée. Elle permet de définir une "table de secours" si on cherche une clé qui n'existe pas dans la table principale.

```lua
local voitureStandard = { moteur = "V6", couleur = "Noir" }
local maVoiturePerso = { couleur = "Rouge" }

-- Si on ne trouve pas de valeur dans 'maVoiturePerso', on cherche dans 'voitureStandard'
setmetatable(maVoiturePerso, { __index = voitureStandard })

print(maVoiturePerso.couleur) -- Sortie : "Rouge" (trouvé localement)
print(maVoiturePerso.moteur)  -- Sortie : "V6" (hérité via la metatable)
```

## Intégration aux Frameworks

### Exemple ESX : L'inventaire
L'inventaire ESX est une table de dictionnaires. Apprendre à la parcourir est essentiel.

```lua
local inventaire = xPlayer.getInventory()

-- 'item' est ici une petite table contenant {name, label, count, etc.}
for _, item in pairs(inventaire) do
    if item.count > 0 then
        -- On utilise les clés de la sous-table
        print("Le joueur a " .. item.count .. " " .. item.label)
    end
end
```

### Exemple QB-Core : Données complexes
QB-Core utilise des tables imbriquées pour les métadonnées de personnage.

```lua
local PlayerData = QBCore.Functions.GetPlayerData()

-- Accès à une valeur dans une table stockée dans une autre table
local niveauFaim = PlayerData.metadata["hunger"]

if niveauFaim < 10 then
    print("Le joueur a faim !")
end
```

## Fonctions Avancées de l'API Table

| Fonction | Usage Pédagogique |
| :--- | :--- |
| `table.concat(t, sep)` | Fusionne une liste de texte en une seule phrase (utile pour les logs). |
| `table.unpack(t)` | Transforme une liste de valeurs en arguments pour une fonction. |
| `table.sort(t)` | Trie les éléments (par ordre alphabétique ou numérique). |

```lua
local coords = {100.5, 20.0, 30.0}
-- unpack(coords) est équivalent à écrire 100.5, 20.0, 30.0
SetEntityCoords(ped, table.unpack(coords))
```

> [!CAUTION]
> **Le piège du Nil** : Si vous insérez `nil` au milieu d'un tableau indexé, l'opérateur `#` (longueur) s'arrêtera à cet endroit et ignorera le reste du tableau. Pour les listes avec des trous, utilisez une boucle `pairs` pour compter manuellement.
