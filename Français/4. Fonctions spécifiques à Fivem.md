# API Natives et Fonctions FiveM

Ce guide se concentre sur les fonctions de base (Natives) fournies par FiveM pour interagir avec l'univers de GTA V. Comprendre ces fonctions est essentiel pour manipuler le monde, les joueurs et les interfaces.

## Qu'est-ce qu'une "Native" ?

Les Natives sont les fonctions "bas niveau" codées par Rockstar Games ou l'équipe FiveM. Elles sont les briques de base de tout script.

*   **Appel Standalone** : Vous appelez directement la native (ex: `GetEntityCoords`).
*   **Appel Framework** : Vous utilisez une fonction "enveloppe" (wrapper) créée par ESX ou QB-Core, qui appelle elle-même plusieurs natives pour vous simplifier la vie.

### La référence indispensable
Toutes les natives sont listées ici : [docs.fivem.net/natives/](https://docs.fivem.net/natives/). C'est le dictionnaire de tout développeur.

## Gestion des Commandes (RegisterCommand)

Utilisez `RegisterCommand` pour créer des interactions via le chat ou la console F8.

### Pourquoi le paramètre 'source' ?
Sur le serveur, `source` est l'identifiant unique du joueur qui a lancé la commande. **Ne jamais oublier de le stocker localement** dès le début de la fonction.

```lua
RegisterCommand("pos", function(source, args, rawCommand)
    -- 'source' est l'ID serveur du joueur
    local coords = GetEntityCoords(PlayerPedId())
    print("Coordonnées envoyées dans la console F8")
end, false)
```

## Événements (Events) : Le Réseau

Les événements permettent de faire voyager des informations entre le **Serveur** (cerveau global) et le **Client** (chaque joueur).

### TriggerServerEvent vs TriggerClientEvent
*   **TriggerServerEvent** : Le client demande quelque chose au serveur (ex: "Je veux acheter ça").
*   **TriggerClientEvent** : Le serveur demande quelque chose à un (ou tous) client(s) (ex: "Affiche ce message sur l'écran").

```lua
-- Côté Client
TriggerServerEvent("ma_ressource:checkLevel")

-- Côté Serveur
RegisterNetEvent("ma_ressource:checkLevel", function()
    local _source = source -- On sécurise la source
    print("Le joueur " .. _source .. " demande son niveau")
end)
```

## Manipulation Opti : Les Vecteurs

Un `vector3` n'est pas juste une liste de chiffres (x, y, z), c'est un **objet mathématique**.

```lua
local ped = PlayerPedId()
local coords = GetEntityCoords(ped)

-- Calcul de distance SIMPLIFIÉ
local destination = vector3(100.0, 200.0, 30.0)
-- L'opérateur '#' sur une soustraction de vecteurs donne la distance en mètres.
local distance = #(coords - destination)

if distance < 5.0 then
    -- Le joueur est très proche
end
```

## Intégration aux Frameworks : Les Notifications

Plutôt que d'utiliser les natives graphiques complexes de GTA V, utilisez les fonctions pré-faites.

### Comparaison par Framework
```lua
-- ESX : Affiche une notification standard en bas à gauche
ESX.ShowNotification("Votre véhicule a été mis en fourrière")

-- QB-Core : Supporte les types (success, error, primary)
QBCore.Functions.Notify("Transaction réussie", "success", 5000)

-- Standalone (Original GTA V)
BeginTextCommandThefeedPost("STRING")
AddTextComponentSubstringPlayerName("Message simple")
EndTextCommandThefeedPostTicker(false, true)
```

> [!IMPORTANT]
> **Le décalage Client/Serveur** : Une native appelée sur le client ne sera pas forcément synchronisée sur le serveur. Pour des actions permanentes (ex: donner un item), passez TOUJOURRS par un événement serveur.
