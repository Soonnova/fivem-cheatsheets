# Syntaxe Lua et Types de Données

Ce guide détaille les fondements du langage Lua dans le complexe spécifique du développement FiveM. Bien que Lua soit considéré comme un langage "léger", sa compréhension profonde est la clé pour créer des scripts GTA V stables, rapides et maintenables.

## Structure et Exécution

Un script Lua dans FiveM est un fichier texte brut portant l'extension `.lua`. Son exécution est **séquentielle** : le moteur lit le fichier de la première à la dernière ligne. Si une erreur de syntaxe survient au début du fichier, tout le reste du script peut échouer.

### Le mot-clé "local" : Pourquoi est-ce vital ?

C'est l'un des points les plus importants pour un développeur FiveM. En Lua, par défaut, toute variable est **globale** à moins d'être explicitement déclarée comme **locale**.

1.  **Performance** : L'accès aux variables locales est plus rapide pour le processeur car elles sont stockées dans des "registres" temporaires plutôt que dans une table globale.
2.  **Sécurité** : Cela empêche un script A de modifier accidentellement une variable du script B si les deux utilisent le même nom.

```lua
-- PRATIQUE RECOMMANDÉE : Portée locale
-- Cette variable n'existe que dans ce fichier ou ce bloc de code.
local messageAccueil = "Bienvenue sur notre serveur"

-- PRATIQUE DÉCONSEILLÉE : Portée globale
-- Cette variable est accessible par TOUS les autres scripts, ce qui crée des conflits.
messageGlobal = "Ceci peut causer des bugs imprévisibles"
```

### Commentaires : Documentez le "Pourquoi"

Les commentaires ne sont pas lus par le jeu. Ils servent à expliquer votre logique à vous-même (dans 6 mois) ou à d'autres développeurs.

```lua
-- Commentaire simple : utile pour une explication courte.

--[[
    Commentaire multi-ligne :
    Idéal pour décrire le fonctionnement global d'un script
    ou pour désactiver temporairement un gros bloc de code.
--]]
```

## Les Types de Données Fondamentaux

Lua identifie automatiquement le type de donnée que vous lui donnez (typage dynamique). Voici ceux que vous utiliserez quotidiennement :

| Type | Description Pédagogique | Exemple Concret |
| :--- | :--- | :--- |
| **Nil** | Représente l'absence totale de valeur. Si vous cherchez une variable qui n'existe pas, Lua renverra `nil`. | `local x = nil` |
| **Number** | Englobe les entiers (1, 2) et les décimaux (1.5). FiveM les utilise pour les IDs, les prix, les coordonnées. | `local prix = 500` |
| **String** | Une suite de caractères (du texte). Toujours entourée de guillemets `" "` ou `' '`. | `local job = "Police"` |
| **Boolean** | Une valeur binaire : soit `true` (vrai), soit `false` (faux). Indispensable pour les conditions. | `local estEnService = true` |
| **Table** | Le conteneur universel de Lua. Elle peut agir comme un simple tableau ou un objet complexe. | `local data = {}` |

### Manipulation des Strings (Texte)

Pour fusionner (concaténer) deux morceaux de texte, Lua utilise l'opérateur `..`. C'est une opération fréquente pour afficher des messages personnalisés.

```lua
local prenom = "John"
local nom = "Doe"
-- On ajoute un espace manuellement entre les deux variables
local nomComplet = prenom .. " " .. nom 
print(nomComplet) -- Affichera "John Doe" dans la console
```

## Les Tables : Le Coeur de Lua

Les tables sont utilisées partout dans FiveM (objets joueurs, configuration de véhicules, listes d'items).

### Le Tableau (Array)
Une liste ordonnée d'éléments. **Attention particulière :** En Lua, on commence à compter à **1**, pas à 0 comme en C# ou Python.

```lua
local metiers = {"Police", "Médecin", "Mécano"}
-- Accéder au premier élément
print(metiers[1]) -- Affichera "Police"
```

### Le Dictionnaire (Object)
Une collection où chaque valeur est associée à une "clé" (un nom).

```lua
local vehicule = {
    modele = "t20",
    plaque = "ABC 123",
    moteurSante = 1000.0
}
-- On accède aux données via le nom de la clé (le "point")
print(vehicule.modele) -- "t20"
```

> [!IMPORTANT]
> Maîtriser les tables est l'étape la plus payante de votre apprentissage. Presque toutes les données complexes renvoyées par FiveM ou vos frameworks (ESX/QB) sont des tables.
