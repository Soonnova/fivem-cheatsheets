# Fondamentaux du NUI (HUD & UI)

Le NUI (New User Interface) est un navigateur web (Chromium) intégré à FiveM. Contrairement à Lua qui gère la logique du jeu, le NUI gère tout l'aspect visuel complexe : menus, HUD, inventaires graphiques.

## Pourquoi un navigateur dans un jeu ?

Lua n'est pas fait pour créer des interfaces graphiques modernes (avec des arrondis, des animations, des ombres). En utilisant le HTML, le CSS et le JavaScript, vous avez accès aux mêmes outils que les développeurs web professionnels pour créer des interfaces magnifiques.

## Le Pont : Communication Lua <-> JS

Le NUI et Lua vivent dans deux mondes différents. Pour qu'ils se parlent, FiveM a créé un "pont" de communication.

### 1. Envoi de données (Lua vers JS)
On utilise `SendNUIMessage`. Imaginez cela comme envoyer une lettre à un destinataire (le navigateur).

**Client (Lua)** :
```lua
-- On envoie une table Lua qui sera automatiquement convertie en JSON
SendNUIMessage({
    type = "OUVRIR_MENU",
    joueur = "John Doe",
    argent = 1500
})
```

**Interface (JavaScript)** :
Le JS "écoute" tout ce qui arrive via l'événement `message`.
```javascript
window.addEventListener('message', (event) => {
    // Les données envoyées par Lua sont dans event.data
    if (event.data.type === "OUVRIR_MENU") {
        document.getElementById("nom").innerText = event.data.joueur;
        document.getElementById("mon-ui").style.display = "block";
    }
});
```

### 2. Retour de données (JS vers Lua)
Le navigateur ne peut pas "appeler" une fonction Lua directement. Il doit faire une requête `POST` vers le nom de votre ressource.

**Interface (JavaScript)** :
```javascript
fetch(`https://${GetParentResourceName()}/fermer_ui`, {
    method: 'POST',
    body: JSON.stringify({ raison: "bouton_quitter" })
});
```

**Client (Lua)** :
On enregistre un "callback" (une fonction qui attend l'appel).
```lua
RegisterNUICallback("fermer_ui", function(data, cb)
    print("L'UI a été fermée car : " .. data.reason)
    SetNuiFocus(false, false) -- On redonne le contrôle du clavier au jeu
    cb('ok') -- Toujours renvoyer une réponse, même vide
end)
```

## Le Focus : Souris et Clavier

Par défaut, quand une UI est ouverte, vos touches contrôlent toujours votre personnage (par exemple, 'Z' fera avancer le joueur au lieu d'écrire dans un champ de texte NUI).

*   `SetNuiFocus(true, true)` : Active le curseur de la souris ET bloque les touches du jeu pour le NUI.
*   `SetNuiFocus(false, false)` : Redonne tout le contrôle au jeu.

## Conseils d'Optimisation (FPS)

1.  **Visibilité** : Une interface invisible (`opacity: 0`) est toujours calculée par le processeur. Utilisez `display: none` pour décharger complètement le rendu quand l'UI est fermée.
2.  **Fréquence** : Ne mettez pas à jour votre HUD (faim, soif) à chaque frame (`Wait(0)`). Une mise à jour toutes les secondes est largement suffisante et préserve les performances.
3.  **Taille des images** : Vos images NUI doivent être légères. Une image de 4Mo ralentira l'ouverture de votre inventaire pour tous les joueurs.

> [!IMPORTANT]
> **GetParentResourceName()** : Utilisez toujours cette fonction en JavaScript plutôt que d'écrire le nom de votre dossier en dur. Cela permet à votre script de fonctionner même si vous renommez le dossier de la ressource.
