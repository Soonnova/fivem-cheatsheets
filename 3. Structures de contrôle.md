# Structures de Contrôle

Ce guide présente les mécanismes permettant de diriger le flux logique dans vos scripts. Une maîtrise des conditions et des boucles est essentielle pour créer des scripts FiveM réactifs, intelligents et performants.

## Comprendre la Logique Conditionnelle (if, elseif, else)

Ces structures permettent à votre script de prendre des décisions selon le contexte (ex: "Si le joueur est dans un véhicule alors...").

### Pourquoi l'ordre des conditions compte ?
Lua vérifie les conditions de manière séquentielle. Dès qu'une condition est vraie (`true`), il exécute le code associé et **ignore** tout le reste du bloc `if`.

```lua
local vitesse = GetEntitySpeed(GetVehiclePedIsIn(PlayerPedId()))

if vitesse > 50.0 then
    -- Si la vitesse est 60, seul ce bloc s'exécute
    print("Vitesse critique")
elseif vitesse > 20.0 then
    -- Ce bloc ne sera JAMAIS lu si la vitesse est supérieure à 50
    print("Vitesse modérée")
else
    print("Vitesse basse ou nulle")
end
```

### Table des Opérateurs Logiques

| Opérateur | Signification Pédagogique |
| :--- | :--- |
| `==` | Vérifie l'égalité stricte. |
| `~=` | Vérifie que deux valeurs sont différentes. |
| `and` | Toutes les conditions doivent être vraies. |
| `or` | Au moins une condition doit être vraie. |
| `not` | Inverse le résultat (vrai devient faux). |

## Les Boucles (Loops) : Travailler à grande échelle

Les boucles sont utilisées pour traiter des listes de données ou pour maintenir des actions répétitives (comme un HUD qui se met à jour).

### Itération de Tables : pairs vs ipairs
C'est une distinction cruciale en Lua :

1.  **ipairs** : Utilisé pour les tableaux numériques ordonnés {1, 2, 3}. Plus performant mais s'arrête au premier index manquant.
2.  **pairs** : Utilisé pour les dictionnaires (clés nommées). Permet de parcourir toute la table, peu importe l'ordre.

```lua
local inventaire = {"Pain", "Eau", "Kit de soin"}

for index, objet in ipairs(inventaire) do
    print("Emplacement " .. index .. " : " .. objet)
end
```

### La Boucle While : Vigilance Maximale
Elle s'exécute tant qu'une condition est vraie. Dans FiveM, on l'utilise souvent dans les threads pour surveiller l'état d'un joueur.

```lua
local estEnVie = true
while estEnVie do
    -- IMPORTANT : Sans Wait(), cette boucle bloquera tout votre Windows/FiveM
    Wait(1000) 
    -- Vérification de l'état ici
end
```

## Intégration aux Frameworks

### Exemple ESX : Filtrer l'inventaire
On utilise `pairs` car l'inventaire ESX est souvent une table complexe d'objets.

```lua
local inventaire = xPlayer.getInventory()

for _, item in pairs(inventaire) do
    -- On vérifie si la quantité est supérieure à 0
    if item.count > 0 then
        print("Le joueur possède " .. item.count .. "x " .. item.label)
    end
end
```

### Exemple QB-Core : Vérification Multi-Critères
Utilisation des opérateurs `and` et `or` pour des permissions complexes.

```lua
local PlayerData = QBCore.Functions.GetPlayerData()

-- "Si le joueur est policier OU ambulancier ET qu'il est en service"
if (PlayerData.job.name == 'police' or PlayerData.job.name == 'ambulance') and PlayerData.job.onduty then
    -- Accès autorisé au vestiaire partagé
end
```

> [!CAUTION]
> **Le piège de la boucle infinie** : Si vous utilisez `while true do` sans un `Wait()` à l'intérieur, votre script demandera 100% de la puissance de votre processeur instantanément, provoquant un gel (freeze) complet du jeu.
